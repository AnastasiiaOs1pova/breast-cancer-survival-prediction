# Прогнозирование выживаемости пациенток с раком груди

Репозиторий содержит ML pipeline для прогноза общей выживаемости пациенток с раком груди на основе клинических и молекулярных данных.

## О проекте

Цель проекта — построить модель, способную прогнозировать общую выживаемость на основе клинических и молекулярных признаков.

Основной акцент сделан на полном ML pipeline:
- анализ структуры и качества данных;
- подготовка и преобразование признаков;
- обучение и сравнение моделей;
- интерпретация предсказаний.

В проекте реализованы:
- разведочный анализ данных (EDA);
- feature engineering;
- сравнение нескольких моделей машинного обучения;
- оценка качества моделей;
- интерпретация результатов с помощью SHAP.

## Используемые технологии

- Python
- Pandas
- NumPy
- Scikit-learn
- Matplotlib / Seaborn
- SHAP

## Dataset

В проекте использован датасет METABRIC с клиническими, молекулярными и мутационными признаками пациенток с раком молочной железы.

- Количество наблюдений: 1904
- Исходное количество признаков: 693
- Целевая переменная: `overall_survival_months`
- Тип задачи: регрессия, прогноз общей выживаемости в месяцах


## Результаты исследования и ограничения постановки задачи

В рамках учебного задания задача была сформулирована как регрессия: модель должна была предсказывать значение `overall_survival_months`, то есть продолжительность общей выживаемости в месяцах.

При этом для реальных задач прогноза выживаемости более корректной постановкой является survival-анализ, так как он учитывает цензурированные наблюдения. Например, если пациентка была поставлена на учёт недавно и на момент сбора данных жива, неизвестно, сколько месяцев она проживёт дальше. Простая регрессия по числу месяцев не учитывает такую неопределённость и может смещать оценки.

В production/research-подходе для подобных задач обычно используют модели survival analysis: Cox Proportional Hazards, Random Survival Forest, Gradient Boosting Survival Models или AFT-модели, а качество оценивают метриками вроде C-index и Integrated Brier Score. В данном проекте survival-анализ не использовался, так как целью было выполнить регрессионную постановку, заданную в учебном проекте.

Лучший результат среди протестированных моделей показал `RandomForestRegressor` после подбора гиперпараметров.
Основная цель проекта — показать end-to-end ML pipeline для медицинских табличных данных: EDA, предобработка, feature engineering, сравнение моделей и интерпретация с помощью SHAP.
Лучший результат показал Random Forest Regressor после подбора гиперпараметров:

Метрики на тестовой выборке для лучшей модели:

- MAE: 55.28 months
- RMSE: 69.03 months
- R²: 0.136

Для интерпретации использовались feature importance и SHAP.
Наиболее значимые признаки для прогноза:

- Nottingham Prognostic Index
- Age at diagnosis
- Number of positive lymph nodes
- Tumor size
- Tumor stage
- PR status
- Selected gene-expression features, including `stat5b`, `bcl2`, `hsd3b7`, `sf3b1`, `pdpk1`

Модель показала клинически ожидаемые связи: худший прогноз чаще связан с высоким Nottingham Prognostic Index, большим числом поражённых лимфоузлов, большим размером опухоли и более старшим возрастом.
